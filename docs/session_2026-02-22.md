# Session Log - 2026-02-22

## Scope Delivered
This session established a deterministic simulator foundation with an interactive CLI watch mode, ruleset-driven behavior, logging artifacts, and API-first roster loading with deterministic fallback.

## Core Architecture and Contracts
- Deterministic contract enforced by:
  - `seed`
  - `model_version`
  - `data_snapshot_id`
  - request context + ruleset checksum
- Simulator behavior externalized into ruleset JSON:
  - default ruleset `rulesets/mlb_2026_regular.json`
  - configurable with `BASEBALL_SIMULATOR_RULESET_PATH`
- Match identity is deterministic:
  - `match_id` derived from deterministic simulation context

## Implemented Components

### API and Domain
- `src/baseball_sim/api/routes.py`
  - health, compare, simulate, and predict endpoints wired.
  - simulation endpoint loads configured ruleset.
- `src/baseball_sim/domain/service.py`
  - simulation service accepts ruleset and ruleset checksum.

### Deterministic Simulation Engine
- `src/baseball_sim/sim/state_machine.py`
  - deterministic plate-appearance state machine.
  - play-level trace support (`PlayTrace`, `GameSimulationTrace`).
  - line-score tracking.
  - deterministic tiebreak resolution at inning cap.
- `src/baseball_sim/sim/rulesets.py`
  - ruleset parsing, validation, checksum, and cached loading.
- `src/baseball_sim/sim/match_id.py`
  - deterministic `match_id` creation.

### CLI Watch Simulator
- `src/baseball_sim/cli/watch.py`
  - live terminal broadcast:
    - scoreboard, inning, outs, bases, line score
    - lineup cards
    - defensive field panel
    - play-by-play stream
  - controls:
    - live: `p` pause, `q` quit
    - paused: `Enter` step, `r` resume, `q` quit
  - configurable delay and manual mode.
- `src/baseball_sim/cli/game_log.py`
  - per-match JSONL play log
  - summary JSON artifact
  - match registry JSONL

### Roster Loading (CLI)
- `src/baseball_sim/cli/roster_loader.py`
  - default loader order implemented:
    1. MLB Stats API roster
    2. deterministic seeded fill for missing slots
    3. full deterministic seeded fallback on API failure/empty payload
  - team name lookup via API with safe fallback.
  - exposes source tags:
    - `api`
    - `api_with_seeded_fallback`
    - `seeded_fallback_api_error`
    - `seeded_fallback_empty_api`
    - `seeded_forced` (`--seeded-rosters-only`)

### Seeder Foundation
- `src/baseball_sim/seeders/roster.py`
  - deterministic synthetic roster generation by `team_id` + `seed`.
